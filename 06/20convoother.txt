  1. Confirmed root cause

  The authenticated user has no public.profiles row. public.handle_new_user()
  (migration 0001) creates a profile only on future auth.users inserts, so
  accounts created before that trigger existed are orphaned. The Meal Logger's
  .single<ProfileRow>() turns "zero rows" into HTTP 406 / PGRST116. Since
  profiles.id is a primary key, "multiple rows" is impossible — zero rows is the
  only explanation. Two amplifiers: toError() only recognized native Errors
  (Supabase errors are plain objects), and profile update().eq() returns no
  error even when it updates zero rows.
